Clonages de templates #

Pré-requis #

Il est supposé que vous avez

  • un compte GitHub enregistré.
  • installé Visual Code Studio avec succès.
  • connaissances de base, comment utiliser Visual Studio Code.
  • installé avec succès μJam extension pour VSCode.
  • Connaissances de base, comment utiliser μJam.

Clonage d'un modèle #

Supposons que vous vouliez créer un site web similaire à un site déjà existant à partir d'un des dépôts de modèles de base μJam ou d'autres. Vous pouvez alors

  • cloner ce dépôt, c'est-à-dire créer une copie locale d'un dépôt GitHub sur votre ordinateur.
  • modifier la structure, le style et le contenu de votre dépôt local.
  • publier votre dépôt local dans un dépôt distant sous votre compte GitHub.

Passons brièvement en revue ces étapes en clonant le dépôt microjam-simple à titre d'exemple.

Démarrez VSCode en mode vide &ndash ; sans fichier ni dossier ouvert. 2. 2. Ouvrez la vue de l'explorateur. Ici, vous êtes proposé à Clone Repository.

  1. Appuyer button Clone Repository et enter le template url.
  1. Il vous sera demandé de sélectionner le répertoire local dans lequel vous souhaitez copier les fichiers modèles.
  2. Après avoir copié avec succès les fichiers modèles, vous recevez une notification vous demandant d'ouvrir le répertoire cloné.
  1. Vous voudrez peut-être ouvrir le dépôt cloné.
  1. La modification de la structure, du style et du contenu de votre dépôt local est décrite ici. Pour publier votre dépôt sur votre compte GitHub lire ici ou ici plus en détail.

Si vous souhaitez créer directement un dépôt GitHub distant à partir d'un dépôt de modèles existant, vous pouvez très bien vous faire guider par ce processus.
